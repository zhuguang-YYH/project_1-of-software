## 📋 详细步骤

### ✏️ 步骤 1：部署初始化函数

**位置**：`cloudfunctions/init_db/`

在微信开发者工具中：

1. 左侧导航栏找到 **云函数** 图标
2. 展开 **cloudfunctions** 目录
3. 找到 **init_db** 文件夹

4. **右键** init_db → 选择 **上传并部署**

   ![上传并部署](./docs/deploy-cloud-function.png)

5. 选择 **云端安装依赖**

6. 等待上传完成（通常 30-60 秒）

7. 底部状态栏显示 ✅ 部署成功

**检查点**：
- [ ] 能看到 "上传成功" 的提示
- [ ] init_db 函数在云函数列表中显示

---

### ⚡ 步骤 2：执行初始化函数

**方式 A：通过小程序控制台（推荐）**

1. 打开小程序任意页面
2. 右键 → **检查元素** 或按 **F12** 打开控制台
3. 找到 **Console** 标签
4. 在底部输入框粘贴以下代码：

```javascript
wx.cloud.callFunction({
  name: 'init_db',
  data: {},
  success: res => {
    console.log('✅ 初始化成功！');
    console.log('返回信息:', res.result);
    alert('数据库初始化成功！数据约 20 条记录');
  },
  fail: err => {
    console.error('❌ 初始化失败');
    console.error('错误信息:', err);
    alert('初始化失败，请查看控制台错误信息');
  }
});
```

5. 按 **Enter** 执行
6. 等待 1-3 秒
7. 应该看到成功提示

**方式 B：通过腾讯云平台**

1. 访问 https://console.cloudbase.tcb.com/
2. 选择环境 `cloud1`
3. 左侧点击 **云函数**
4. 找到 `init_db` 函数
5. 点击 **测试** 按钮
6. 点击 **执行**
7. 查看执行结果

**执行成功标志**：
```json
{
  "code": 0,
  "message": "数据库初始化成功"
}
```

**执行失败标志**：
```json
{
  "code": -1,
  "message": "数据库初始化失败: ...",
  "error": "..."
}
```

## ⚠️ 常见问题排查

### Q1：执行 init_db 后显示"Function not found"
**A：** init_db 函数未部署成功
- 确保在云函数中看到 init_db 函数
- 重新部署：右键 init_db → 上传并部署 → 云端安装依赖
- 等待部署完全完成

### Q2：执行成功但数据库集合仍为空
**A：** 可能是权限或初始化逻辑问题
- 查看云函数执行日志（腾讯云平台 → 函数日志）
- 检查是否有错误信息
- 尝试清空集合后重新执行

### Q3：只看到部分集合或数据
**A：** 初始化过程中某个步骤失败
- 查看控制台完整输出信息
- 根据错误信息检查对应集合
- 手动创建缺失的集合（如需要）

### Q4：如何修改初始数据？
**A：** 编辑源文件后重新部署
1. 打开 `cloudfunctions/init_db/index.js`
2. 修改 users、activities 等数组数据
3. 保存文件
4. 重新上传部署
5. 清空旧数据后重新执行初始化

### Q5：能否多次执行初始化？
**A：** 可以，但建议：
- 初始化具有去重逻辑，相同数据不会重复添加
- 如需完全重新初始化，先手动删除集合
