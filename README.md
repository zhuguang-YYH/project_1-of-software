# NK推协侦探管理系统

本项目是基于微信开发者工具开发的微信小程序，后端使用微信云开发云函数与云数据库。当前版本围绕 NK 推理协会的社团运营场景，实现了每日谜题、积分体系、排行榜、活动报名、物资借阅、积分兑换、事件委托、Dud 关键词回复、个人名片、反馈留言和小程序内后台管理等核心模块。

## 当前版本已实现功能

### 用户与个人中心

- 微信授权登录与自动登录。
- 用户基础资料维护，包括昵称、头像、签名等。
- 个人名片展示组件，排行榜等个人展示入口可跳转名片详情。
- 个人页聚合活动、借阅、兑换、积分、委托、反馈、设置和后台入口。

### 每日谜题与积分

- 每日谜题获取、选项展示和答题提交。
- 每人每日一次答题校验。
- 答题正确后增加积分并写入积分流水。
- 积分账户拆分为 `total_points`、`available_points`、`frozen_points`、`used_points`。
- 积分明细页支持查看流水和分页加载。

### 排行榜

- 首页/排行榜页展示前三名与完整榜单。
- 支持同分并列排名。
- 生成 `ranking_snapshots` 排行榜快照。
- 排行榜用户项可打开个人名片详情。

### 活动报名

- 活动列表与我的活动。
- 活动容量、报名状态、报名理由和取消报名流程。
- 后端校验重复报名、容量上限与取消截止时间。

### 物资与剧本杀借阅

- 统一库存集合 `inventory_items` 管理书籍、剧本杀和兑换商品。
- 可借物资、剧本杀列表和我的借阅记录。
- 借阅申请、传递中取消、借出/归还状态流转的后端支持。
- 管理后台可创建库存物资并处理借阅记录。

### 积分兑换

- 兑换商品列表、库存和所需积分展示。
- 兑换时扣减可兑换积分并冻结相应积分。
- 写入 `exchange_records` 兑换记录。
- 管理后台可创建兑换商品并处理兑换状态。

### 事件委托

- 用户发布委托并冻结奖励积分。
- 用户领取委托、标记完成。
- 发布者确认并分配奖励积分。
- 支持官方置顶委托和委托奖励流水。

### Dud 关键词回复

- Dud 对话页面。
- 关键词规则匹配。
- 管理后台可维护关键词、回复内容、匹配方式、优先级和启用状态。

### 推荐内容

- 首页推荐内容摘要。
- 推荐列表与详情。
- 后台可发布和上下架推荐内容。

### 反馈留言

- 用户提交反馈，支持匿名。
- 用户查看自己的反馈记录。
- 管理后台查看反馈并填写处理备注。

### 管理后台

- 基于用户 `role=admin` 的权限校验。
- 数据概览统计。
- 谜题、活动、库存物资、兑换商品、推荐内容、Dud 规则、反馈、系统设置等管理入口。
- 关键操作写入 `admin_logs`。

### 数据库对齐

- 主要业务字段已逐步对齐需求文档中的 snake_case 设计。
- `init_db` 云函数支持初始化核心集合。
- `init_db` 支持通过 `cleanupLegacyFields: true` 清理旧字段。
- 已将借阅、兑换、反馈、委托、排行榜等模块从旧字段调用逐步迁移到新字段。

## 初始化与部署

1. 使用微信开发者工具打开项目根目录。
2. 在云开发控制台中创建需要的集合。
3. 部署所有 `cloudfunctions/` 下的云函数。
4. 调用 `init_db` 云函数初始化数据。
5. 如需清理旧字段，调用 `init_db` 时传入：

```json
{
  "cleanupLegacyFields": true
}
```

## 当前仍需开发或优化

- 页面视觉资源仍需替换为正式美术素材，包括图标、背景、卡片装饰、默认头像、商品/活动封面等。
- 部分页面为了规避编码问题，按钮和状态文案暂时使用英文或较简短文案，后续可统一做中文文案与 i18n/编码整理。
- 云函数尚未全面使用数据库事务；兑换、委托奖励、借阅状态等高并发场景后续建议改为事务或更严格的原子更新。
- 后台管理仍可继续细化，例如筛选、搜索、批量处理、状态审计和操作回滚说明。
- `init_db` 仍保留旧字段清理清单，这是为了清理历史数据；数据库清理完成后可进一步瘦身。
- 还缺少自动化测试和端到端测试。
- 需要在真实微信环境中做完整联调，包括登录、云函数权限、数据库权限规则和云存储上传。
